# อ้างอิง: หน้าต่าง (Modals) และปุ่มทั้งหมด

หน้านี้รวบรวม **หน้าต่าง (modal/dialog)** และ **ปุ่ม** ทุกชนิดที่พบในระบบ พร้อมความหมาย
วิธีเปิด/ใช้งาน และบทที่แสดงตัวอย่างจริง. ใช้เป็นคู่มืออ้างอิงเร็ว — รายละเอียดการทำงาน
แบบ step-by-step อยู่ในบทที่ 1–5.

> **หลักการตั้งชื่อปุ่ม:** ปุ่มสีส้ม/หลัก = การกระทำสำคัญ (ออกเอกสาร/โพสต์/อนุมัติ);
> ปุ่มขอบ/โปร่ง = บันทึกร่าง; ปุ่มจาง = ยกเลิก/ย้อนกลับ. ระบบ **ซ่อนปุ่มที่ผู้ใช้ไม่มีสิทธิ์**
> อัตโนมัติ (ดู "สิทธิ์การมองเห็น (RBAC UI Guide)").

## 1. หน้าต่าง (Modals / Dialogs)

### 1.1 เลือกลูกค้า / เลือกผู้ขาย (Entity Picker)
- **เปิดด้วย:** ปุ่ม **"+ เลือกลูกค้า"** หรือ **"+ เลือกผู้ขาย"** ในส่วน ① ของหน้าสร้างเอกสาร
  (เมื่อเลือกแล้วปุ่มเปลี่ยนเป็น **"เปลี่ยน"** สำหรับสลับราย).
- **ภายใน:** ช่องค้นหา (พิมพ์ชื่อ/รหัส/เลขผู้เสียภาษี) → รายการผลลัพธ์ (กดเพื่อเลือก).
- **แสดงใน:** ทุกหน้าสร้างเอกสาร · สลับชนิดผู้ซื้อใน **04.09** · สลับชนิดผู้ขายใน **05.05**.

### 1.2 เลือกสินค้า / บริการ (Product Search)
- **เปิดด้วย:** ปุ่มแว่นขยาย 🔍 **"เลือกจากรายการ"** ท้ายช่องรายละเอียดของแต่ละบรรทัด.
- **ภายใน:** ช่องค้นหา · รายการสินค้าพร้อม **ป้ายประเภท** (🟦 สินค้า / 🟪 บริการ /
  🟧 ยกเว้น VAT) + ราคาตั้งต้น · ปุ่ม **"+ สร้างสินค้า/บริการใหม่"** · ปุ่ม **"ยกเลิก"**.
- **เลือกแล้วได้:** ชื่อ + ราคา + หน่วย + ประเภท (ประเภทกำหนด VAT อัตโนมัติ).
- **แสดงใน:** **04.08** (พร้อมการเลือกบริการ/สินค้ายกเว้น VAT).

### 1.3 สร้างสินค้า/บริการใหม่ (Quick Create)
- **เปิดด้วย:** ปุ่ม **"+ สร้างสินค้า/บริการใหม่"** ในหน้าต่างเลือกสินค้า (ข้อ 1.2).
- **ใช้เมื่อ:** สินค้ายังไม่มีใน master — สร้างทันทีแล้วเลือกใช้เลย (seed ชื่อจากที่พิมพ์ค้างไว้).

### 1.4 ยืนยันการบันทึกเอกสาร (Post Confirm)
- **เปิดด้วย:** กดปุ่ม **"บันทึกเอกสาร"** / **"บันทึก (Post)"** บนเอกสารที่ตรึงถาวรเมื่อโพสต์
  (ใบกำกับภาษี, ใบเสร็จ, ใบลดหนี้/เพิ่มหนี้, ใบกำกับภาษีซื้อ, ใบแจ้งหนี้).
- **ภายใน:** สรุปยอด + คำเตือน **"แก้ไขไม่ได้อีกหลังบันทึก"** · ปุ่ม **"ยกเลิก"** + ปุ่ม
  **"ยืนยันบันทึก"** (สีเตือน). หัวข้อกล่องเปลี่ยนตามชนิดเอกสาร.
- **แสดงใน:** **04.04** (ใบกำกับภาษี) · **04.05** (ใบเสร็จ) · **04.06/04.07** (CN/DN) · **05.02** (ภาษีซื้อ).

### 1.5 หน้าต่างเพิ่ม/แก้ไขข้อมูลหลัก (Master-data modal)
- **รูปแบบ:** กล่องกลางจอ (DaisyUI `.modal-box`) — มีฟอร์มกรอก + ปุ่ม **"บันทึก"** / **"ยกเลิก"**.
- **พบใน:** หน่วยธุรกิจ (**02.01**) · สินค้า/บริการ (**02.02**) · ประเภทภาษีหัก ณ ที่จ่าย
  (**02.03**) · API Keys (**02.04**) · พนักงาน (**03.05**).
- *(ลูกค้า/ผู้ขายใช้หน้าเต็ม ไม่ใช่ modal — ดู 03.01/03.02.)*

### 1.6 กล่องยืนยัน (Confirm dialog)
- **ใช้กับ:** การกระทำที่ย้อนยาก เช่น ปิดใช้งาน/ลบข้อมูลหลัก, ยกเลิกเอกสาร — มีปุ่ม
  **"ยืนยัน"** + **"ยกเลิก"** เสมอ.

## 2. ปุ่มในหน้า "สร้างเอกสาร" (Create forms)

ปุ่มทั้งหมดอยู่แถบบนขวาของฟอร์ม (นอกพื้นที่กรอก) — กดแล้วระบบตรวจความถูกต้องก่อนเสมอ.

| ปุ่ม | ความหมาย | เอกสารที่ใช้ |
|---|---|---|
| **ยกเลิก** | ทิ้งร่าง กลับหน้ารายการ | ทุกเอกสาร |
| **บันทึกร่าง** | เก็บเป็นฉบับร่าง (ยังไม่ออกเลข) แก้ได้ | ใบเสนอราคา · ใบกำกับภาษี · ใบแจ้งหนี้ · ใบส่งของ · ใบกำกับภาษีซื้อ |
| **บันทึก** | สร้างเอกสาร (PO/PV ออกเลขตอนอนุมัติทีหลัง) | ใบสั่งซื้อ · ใบสำคัญจ่าย |
| **ออกใบเสนอราคา / ออกใบส่งของ / ออกใบแจ้งหนี้** | ออกเอกสาร (ออกเลข + เปลี่ยนสถานะเป็น "ส่ง/ออกแล้ว") | ใบเสนอราคา · ใบส่งของ · ใบแจ้งหนี้ |
| **ยืนยันสั่งขาย** | ยืนยันใบสั่งขาย (ออกเลข) | ใบสั่งขาย |
| **บันทึกเอกสาร / บันทึก (Post)** | โพสต์ → ออกเลขถาวร + **ตรึงข้อมูล** (ผ่านกล่องยืนยัน 1.4) | ใบกำกับภาษี · ใบเสร็จ · ใบลดหนี้/เพิ่มหนี้ · ใบกำกับภาษีซื้อ |

## 3. ปุ่มบน "หน้ารายละเอียดเอกสาร" (Detail pages)

ปุ่มที่ปรากฏขึ้นกับ **สถานะ** ของเอกสาร และ **สิทธิ์** ของผู้ใช้.

### 3.1 ใบเสนอราคา → ใบสั่งขาย → ใบส่งของ
- **ลูกค้าตอบรับ / ลูกค้าปฏิเสธ** — บันทึกผลการตอบกลับใบเสนอราคา.
- **แปลงเป็นใบสั่งขาย / สร้างใบส่งของ** — แปลงไปขั้นถัดไปของสายงานขาย.
- **ยกเลิก<เอกสาร>** — ยกเลิกเอกสารที่ยังไม่ปิด.

### 3.2 ใบสั่งซื้อ (PO) — ดู **05.01**
- **อนุมัติ** (`po-approve`) — Draft → Approved, **ออกเลข PO-NNNN ตอนนี้**.
- **สร้างใบสำคัญจ่าย** (`po-create-pv`) — สร้าง PV อ้างอิง PO.
- **ส่ง PO ให้ vendor** · **ปิด** (`po-close`) · **ยกเลิก** (`po-cancel`).

### 3.3 ใบกำกับภาษี (Tax Invoice) — ดู **04.04 / 04.05**
- **บันทึก (Post)** (`ti-post-action`) — โพสต์ใบที่ยังเป็นร่าง (ผ่านกล่องยืนยัน).
- **สร้างใบเสร็จ** — ออกใบเสร็จอ้างอิงใบนี้ (เติมลูกค้า/ยอดให้, แสดงเมื่อ "โพสต์แล้ว + ยังไม่ชำระ").
- **พิมพ์ / PDF** · **ดาวน์โหลด XML (e-Tax)** · **ส่งอีเมลอีกครั้ง** (e-Tax — เฉพาะกิจการ VAT).

### 3.4 ใบแจ้งหนี้ (Invoice/BillingNote) — ดู **04.03**
- **ออกใบกำกับภาษี** — ออกใบกำกับภาษีจากใบแจ้งหนี้ (กิจการ VAT).
- **ยืนยันชำระเงินครบแล้ว** — ปิดยอดเมื่อรับเงิน.

### 3.5 ใบสำคัญจ่าย (PV) — ดู **05.03**
- **อนุมัติ** (`pv-approve`) — Draft → Approved.
- **บันทึก (Post)** (`pv-post`) — Approved → Posted (ลงบัญชี + ออก 50ทวิ; ตรึงถาวร).
- **สร้างใบกำกับภาษีซื้อ** (`pv-create-vi`).

### 3.6 ใบเสร็จ / ใบลดหนี้ / ใบเพิ่มหนี้ / ภาษีซื้อ
- เอกสารเหล่านี้ **ตรึงถาวรหลังโพสต์** — หน้ารายละเอียดมีแค่ **พิมพ์ / PDF** (แก้ไม่ได้;
  แก้ไขใบกำกับภาษี = ออกใบลดหนี้/เพิ่มหนี้แทน ดู 04.06/04.07).

### 3.7 หน้าภาษี (Tax filings) — ปุ่ม "ดาวน์โหลด PDF" แบบที่ระบบกรอกให้
หน้ายื่นภาษีมีปุ่ม **"ดาวน์โหลด PDF"** ที่ให้ไฟล์ **แบบของกรมสรรพากรที่ระบบกรอกให้แล้ว**
(พิมพ์ออกยื่นได้เลย — ระบบ **ไม่ส่งให้สรรพากรแทน**). กดข้างปุ่ม "แสดงตัวอย่าง/ยืนยัน" หลังเลือกงวด.
- **ดาวน์โหลด PDF (ภ.พ.30)** — หน้า **/reports/pnd30** (ดู **07.07**). VAT return ที่กรอกครบ.
- **ดาวน์โหลด PDF (ภ.ง.ด.3 / ภ.ง.ด.53)** — หน้า **/tax-filings** (ดู **07.08 / 07.09**). WHT
  หน้าหลัก + **ใบแนบ** (รายผู้ถูกหักทีละราย); ระบบเลือก ภ.ง.ด.3 (บุคคลธรรมดา) / ภ.ง.ด.53
  (นิติบุคคล) ให้ตามประเภทผู้รับเงิน.
- **ดาวน์โหลด PDF (ภ.ง.ด.54)** — หน้า **/tax-filings/pnd54** (ดู **07.10**). WHT จ่ายต่างประเทศ
  ตาม ม.70 (กรอกหัวฟอร์ม + เครื่องหมาย ม.70/ยื่นปกติ).
- เทียบกับ **พิมพ์ / PDF** ในข้อ 3.1–3.6 ที่พิมพ์ *ตัวเอกสาร* (ใบกำกับภาษี ฯลฯ) — ข้อนี้คือ
  **แบบยื่นภาษีของกรมสรรพากร** ที่ระบบกรอกยอดให้.

## 5. ความแตกต่างตามสถานะ VAT ของบริษัท

เมนูและปุ่มบางส่วน **ปรับอัตโนมัติ** ตามว่าบริษัทจด VAT หรือไม่ (ดู **04.10**):

- **กิจการไม่จด VAT:** ซ่อนเมนู/ปุ่มที่เกี่ยวกับ **ใบกำกับภาษี · ใบลดหนี้ · ใบเพิ่มหนี้ ·
  e-Tax** (ออกไม่ได้ตาม ม.86); ใบเสร็จใช้โหมด "เงินสด/อ้างอิงใบแจ้งหนี้" แทน "อ้างอิงใบกำกับภาษี".
- **กิจการจด VAT:** มีครบทุกเมนู + ปุ่ม e-Tax (XML/ส่งอีเมล).

> ปุ่มสลับโหมด VAT **ไม่มี** ในหน้าผู้ใช้ทั่วไป — สถานะ VAT เป็นข้อมูลของบริษัท ตั้งโดย
> ผู้ดูแลระบบสูงสุดเท่านั้น (§4.6, ดูบทที่ 0).
