# บทที่ 1 — เริ่มต้นใช้งาน TEAS

ยินดีต้อนรับสู่ TEAS — ระบบบัญชีสำหรับองค์กรไทยที่ออกแบบให้สอดคล้องกับ
ประมวลรัษฎากร, พระราชบัญญัติการบัญชี พ.ศ. 2543, มาตรฐานบัญชีไทย (TFRS),
และ PDPA.

ก่อนเริ่มอ่านบทนี้ คุณควรมี:

- บัญชีผู้ใช้ (user account) ที่ผู้ดูแลระบบสร้างให้แล้ว
- URL ของระบบ (เช่น `https://teas.your-company.example` หรือ `http://localhost:3000` สำหรับ dev)
- Browser รุ่นใหม่ — Chrome 78+, Edge 18+, Firefox 70+, Safari 13+

## แนวคิดที่ควรรู้ก่อนเริ่ม

### ความปลอดภัยของ session

TEAS ใช้ **httpOnly cookie** เก็บ session token — Browser จะไม่ให้ JavaScript
ของหน้าเว็บอ่าน token นี้ได้ (BFF cookie pattern). หมายความว่าแม้ว่าจะมี
malicious script รั่วเข้ามาในหน้า เว็บ session ของคุณก็ไม่ถูกขโมยง่าย ๆ.

### หลายบริษัทใน 1 ระบบ (Multi-Company)

ระดับ database TEAS รองรับ multi-tenant อยู่แล้ว — ทุก transaction
จะถูก tag กับ `company_id` ของบริษัทที่ session กำลังใช้งาน
(ใช้ PostgreSQL RLS + EF global filter เป็น 2 ชั้น).

User ที่มี role ใน 1 บริษัท จะเห็นเฉพาะข้อมูลของบริษัทนั้นเสมอ.
เวอร์ชันปัจจุบันรองรับ "1 user = 1 default company" — UI สำหรับสลับ
บริษัทระหว่าง session ยังไม่อยู่ใน sidebar (อยู่ใน Phase 2 backlog).
หากต้องเปลี่ยน default company ของ user ให้ admin แก้ผ่าน User
Management หรือผ่าน DB seed.

### Audit Trail ทุก action

ทุก mutation (สร้าง / แก้ไข draft / Post / ลบ) ถูกบันทึกใน activity log
พร้อม timestamp + user + details. คุณไม่สามารถลบ audit log ได้ (กฎหมาย
บัญชี ม.14 — append-only).

### Document Immutability หลัง Post

เมื่อ document (TI, RC, CN, DN, JV, ฯลฯ) ถูก Post แล้ว — ห้ามแก้ไข
ตามกฎหมาย. การแก้ผิดต้องผ่าน Credit Note + reissue เท่านั้น.
รายละเอียดอ่านบทที่ 3 ใบลดหนี้.
