# TEAS Connect (MCP) — OAuth 2.1 Authorization Server — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let Claude Desktop/Mobile native connectors, Codex CLI, and Gemini Spark connect to TEAS Connect (`/mcp`) via standard MCP OAuth 2.1 + PKCE, reusing the existing X-Api-Key tenant/scope machinery unchanged.

**Architecture:** Add OpenIddict 7.5.0 as an OAuth 2.1 Authorization Server on the existing `.NET 10` API. Its EF stores live in the existing `AccountingDbContext` under a new `oauth` schema (one migration). The load-bearing insight: an issued OAuth access token carries the SAME claims the X-Api-Key handler emits (`company_id`, `branch_id`, `is_api_key="true"`, `scopes` CSV, `ClaimTypes.Name`) → `HttpTenantContext` → RLS → `apiperm:*` gates → MCP tools all work unchanged. OAuth is an alternate front door to the same machinery. `/mcp` accepts Bearer XOR X-Api-Key. `*.post`-class scopes are structurally never grantable to an OAuth token.

**Tech Stack:** OpenIddict.AspNetCore 7.5.0 + OpenIddict.EntityFrameworkCore 7.5.0 · EF Core 10 / Npgsql · ASP.NET Core Minimal APIs · Next.js 15 (passthrough + consent page) · xUnit + FluentAssertions + Testcontainers/`TEAS_TEST_PG`.

## Global Constraints

**These bind every task. The reviewer's attention lens. Copied verbatim from spec §6b (BINDING Codex must-fixes) + §6 (RFC 8707).**

- **`is_api_key = "true"`** — load-bearing: `PermissionHandler` (`PermissionRequirement.cs:17-43`) only reads the CSV `scopes` when `is_api_key` is set; without it it falls back to JWT `perm` claims and **denies every MCP tool call**.
- **`company_id`, `branch_id`** — positive + valid + related; **reject the token if either is 0** (silent-zero caused duplicate doc-number sequences). Hard-reject at validation, not just at grant.
- **`sub`** = the immutable **numeric TEAS user id**; **`ClaimTypes.Name`** = actor name.
- **CSV `scopes`** = the granted MCP set (normalized), authoritative from the server principal — never browser-posted.
- **NEVER emit `is_super_admin=true`** — it bypasses the EF tenant filter (`AccountingDbContext.cs:140-145`) AND sets the Postgres super-admin RLS flag (`TenantMiddleware.cs:31-38`). Hard-exclude from every OAuth principal.
- **No `api_key_id`** on OAuth tokens (OAuth ≠ api key); provenance = user `sub` + `client_id` + token id.
- **Authorization scheme:** `apiperm:*` policies hard-wire `.AddAuthenticationSchemes(ApiKey)` (`PermissionPolicyProvider.cs:28-37`). Refactor `PermissionPolicyProvider` + `ApiKeyOnlyPolicy` to accept BOTH `ApiKey` and the OpenIddict validation scheme. `/mcp` selects exactly ONE credential (ApiKey XOR Bearer); reject requests carrying both. Centralize `/mcp` 401 so Bearer gets `WWW-Authenticate: Bearer resource_metadata="{BaseUrl}/.well-known/oauth-protected-resource"` (not the ApiKey JSON body).
- **Consent bridge:** do NOT round-trip `client_id`/`redirect_uri`/PKCE/`resource`/scopes/company through editable browser fields. Use OpenIddict's server-side authorization-request cache keyed on an opaque one-time handle bound to the TEAS session; the accept endpoint re-enters the validated OpenIddict pipeline and finishes with `SignIn(principal, OpenIddictServerAspNetCoreDefaults.AuthenticationScheme)` (deny = `Forbid(...)`, never manual code construction). BFF accept needs antiforgery + SameSite + Origin/Referer checks. OpenIddict validates the exact registered `redirect_uri` + PKCE **S256** before consent.
- **Scope guard (positive allowlist):** define `McpAllowedScopes` (read+create+manage only, no `.post`-class). `granted = request.GetScopes() ∩ McpAllowedScopes`; reject unknown/`.post` as `invalid_scope`. `principal.SetScopes(granted)` before `SignIn`. Token endpoint **preserves** scopes from the code/refresh principal, never rebuilds; re-check no-`.post` on refresh. Restrict each MCP application via `OpenIddictApplicationDescriptor.Permissions`; DCR assigns fixed MCP permissions **server-side** (clients can't self-grant `.post`).
- **Tokens/compliance:** access token **5–15 min**; **rotating refresh** with reuse detection + short absolute lifetime (~1 workday). `company_id` baked at grant — switching the web UI's active company MUST NOT retarget an existing token; refresh keeps the original company. On every refresh revalidate user status + company membership/activation + revocation; revoke the token family on membership removal / credential reset / refresh reuse. Partition `/mcp` Bearer rate-limit by token/client (not the shared `__no_api_key` bucket, `Program.cs:156-170`).
- **RFC 8707 `resource`:** clients pass `resource` (the MCP server URL) on authorize + token. The AS validates it and encodes the target in the token `aud`; the resource server checks `aud` = its own `/mcp` URL.
- **Audit:** every issuance → `audit.activity_log`: user `sub` + actor + `client_id` + auth/token id + company.
- **Support BOTH** DCR (RFC 7591) and manual pre-registration. PKCE **S256** required. MCP spec rev = 2025-11-25.
- **Env/footguns (§6):** run `dotnet`/`ef` from `W:`; kill :5080 before a full build; **NEVER `dotnet ef … --no-build` after entity edits** (build the solution first, then `ef` WITH build); integration tests use `TEAS_TEST_PG` (+ `TEAS_REPO_ROOT` for RBAC tests) and a fresh company via `TestCompanyFactory` (never flip company 1); FE `tsc --noEmit` 0 and never `next build` while `next dev` runs.
- **Compliance:** `TreatWarningsAsErrors=true` (build must be 0 warnings). CPM on (`ManagePackageVersionsCentrally=true`) → package versions ONLY in `Directory.Packages.props`, csproj `PackageReference` carry no `Version`. **Do NOT run the prod migration / deploy** — Ham runs those.

```csharp
namespace Accounting.Application.Abstractions;

/// <summary>
/// The read+create+manage scope set an MCP OAuth token may be granted. Mirrors the mcp-kind
/// X-Api-Key default scopes. *.post-class scopes are structurally excluded (an agent cannot post).
/// </summary>
public static class McpScopes
{
    // Mirror of the mcp-kind key's MCP_DEFAULT_SCOPES (frontend api-keys page).
    public static readonly IReadOnlyList<string> All = new[]
    {
        "sales.tax_invoice.read",   "sales.tax_invoice.create",
        "sales.receipt.read",       "sales.receipt.create",
        "sales.quotation.read",     "sales.quotation.create",
        "master.customer.read",     "master.customer.manage",
        "master.product.read",      "master.product.manage",
        "master.vendor.manage",
        "purchase.purchase_order.read",  "purchase.purchase_order.create",
        "purchase.vendor_invoice.read",  "purchase.vendor_invoice.create",
        "purchase.payment_voucher.read", "purchase.payment_voucher.create",
        "sys.system_info.read",
    };

    // Mirror of ApiKeyService.McpForbiddenSuffixes (defense-in-depth invariant check).
    public static readonly string[] ForbiddenSuffixes =
        [".post", ".approve", ".issue", ".send", ".void", ".cancel", ".reject"];

    /// <summary>Granted = requested ∩ All (rejects unknown + any *.post-class by construction).</summary>
    public static IReadOnlyList<string> Normalize(IEnumerable<string> requested) =>
        requested.Where(s => All.Contains(s, StringComparer.Ordinal)).Distinct(StringComparer.Ordinal).ToArray();
}
```

```csharp
var identity = new ClaimsIdentity(OpenIddictServerAspNetCoreDefaults.AuthenticationScheme);
identity.SetClaim(Claims.Subject, userId.ToString());          // sub = numeric TEAS user id
identity.SetClaim(ClaimTypes.Name, actorName);
identity.SetClaim(TenantClaims.CompanyId, companyId.ToString());
identity.SetClaim(TenantClaims.BranchId, branchId.ToString());
identity.SetClaim(TenantClaims.IsApiKey, "true");             // load-bearing
identity.SetClaim(TenantClaims.Scopes, string.Join(',', granted));
// NEVER is_super_admin; NEVER api_key_id.
var principal = new ClaimsPrincipal(identity);
principal.SetScopes(granted);
principal.SetResources(request.Resources.ToArray());          // RFC 8707 → aud
principal.SetDestinations(static claim => claim.Type switch
{

        or TenantClaims.IsApiKey or TenantClaims.Scopes => [Destinations.AccessToken],
    _ => [Destinations.AccessToken],
});
return SignIn(principal, OpenIddictServerAspNetCoreDefaults.AuthenticationScheme);
```
